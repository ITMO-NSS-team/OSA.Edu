# OSA.Edu

OSA.Edu — система автоматизированной предварительной проверки ВКР и диссертаций по набору формальных, структурных и содержательных требований.

## Возможности

- загрузка документов PDF и DOCX;
- автоматическое извлечение текста;
- построение структурной карты документа;
- ручное подтверждение и редактирование структуры;
- проверка документа по набору правил;
- формальные и структурные проверки;
- содержательные проверки с использованием LLM;
- привязка найденных замечаний к фрагментам документа;
- формирование итогового отчёта;
- экспорт отчёта в Markdown и JSON.

## Как работает система

1. Пользователь загружает PDF или DOCX.
2. Из документа извлекается текст.
3. Текст разбивается на блоки с привязкой к страницам.
4. LLM определяет структуру документа: название, введение, цель, задачи, положения, главы, выводы, заключение и список литературы.
5. Пользователь подтверждает или исправляет структурную карту.
6. Каждому правилу назначается подходящая область документа и способ проверки.
7. Формальные проверки выполняются программными детекторами, содержательные — с помощью LLM.
8. Для найденных замечаний система сохраняет доказательства: цитату, страницу и идентификатор блока.
9. Результаты объединяются в итоговый отчёт.

## Установка

Установите Node.js-зависимости:

```bash
npm ci
```

Создайте Python-окружение.

### macOS / Linux

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### Windows

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Создайте файл `.env` на основе `.env.example`.

### macOS / Linux

```bash
cp .env.example .env
```

На Windows можно создать копию `.env.example` вручную и назвать её `.env`.

Укажите API-ключ OpenRouter:

```env
OPENROUTER_API_KEY=your_api_key
```

## Запуск

Запустите frontend и backend:

```bash
npm run dev
```

После запуска:

- интерфейс: `http://127.0.0.1:5173`
- API: `http://127.0.0.1:8787`

## Правила

В проекте доступны два профиля проверки:

- `core` — основной набор правил;
- `full` — расширенный набор правил.

Также пользователь может добавить собственные требования перед запуском проверки.

## Статусы проверки

Правило может получить один из статусов:

- `pass` — нарушение не найдено;
- `violation` — найдено подтверждённое нарушение;
- `uncertain` — система не может сделать однозначный вывод;
- `not_applicable` — правило не применяется к документу;
- `not_checked` — проверка не была выполнена.

## Ограничения

Точность проверки зависит от качества извлечения текста из исходного документа и работы выбранной LLM.

Содержательные фрагменты документа могут передаваться внешнему LLM-провайдеру при выполнении семантических проверок.

## Дисклеймер

OSA.Edu предоставляет рекомендации для последующей экспертной проверки. Автоматический отчёт не является официальным заключением, не определяет научную ценность работы и не заменяет решение научного руководителя, нормоконтролёра, рецензента или аттестационной комиссии.
